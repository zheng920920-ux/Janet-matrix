# AI Matrix Dashboard 2.0 Review Brief

请你帮我 review 一个移动端优先的个人投资管理原型。

这个系统不是公开投顾平台，只是个人投资记录和辅助判断工具。所有建议都必须遵守：

> 仅供个人参考，不构成投资建议。

## 产品定位

Personal Investment OS，个人投资操作系统。

重点不是展示更多基金数据，而是每天帮助我回答：

- 今天我的投资组合怎么样？
- 今天为什么赚/亏？
- 我的组合是否健康？
- 哪些风险今天最重要？
- 哪些基金需要继续持有、观察、进一步比较？
- 今天应该继续定投、暂停新增，还是只观察？

## 技术栈

- Next.js
- TypeScript
- Tailwind CSS
- shadcn/ui 风格组件
- ECharts
- 当前阶段使用 mock data
- 移动端优先，适配手机浏览器/PWA

首页第一屏现在只保留：

## Morning Brief

Morning Brief 是首页最重要模块，控制在 3-5 行。

它回答：

- 今天市场发生了什么？
- 为什么今天盈利/亏损？
- 今天建议做什么？

示例内容：

```text
昨日纳斯达克100上涨 1.12%，美元兑人民币上涨 0.18%。
组合今日估算盈利，主要由纳指/QDII贡献。
QDII今日收益为估算值，不建议根据单日估算频繁操作。
今日动作：黄金暂停新增，半导体基金加入观察名单。
```

## 今日重点风险

风险不再平铺，改成分级：

- 高风险
- 中风险
- 普通提醒

示例：

```text
高风险：基金规模偏小
中风险：黄金仓位偏高 / 未满7天
普通提醒：QDII估算收益 / Mock Data
```

## Investment Health Score

首页不再重点展示每天变化很小的资产饼图，而是展示组合健康评分：

```text
Investment Health Score
86 / 100
```

评分依据包括：

- 黄金仓位是否偏高
- 基金是否长期跑输
- 基金规模是否偏小
- 现金比例是否不足
- 是否存在未满7天持仓
- QDII占比是否合理

资产健康度示例：

```text
基金：观察
黄金：偏高
股票：正常
现金：略低
```

## 为什么今天盈利/亏损

首页新增收益解释模块，但放在“查看持仓”按钮之后，避免抢第一屏。

示例：

```text
纳指/QDII +580
原因：纳斯达克100上涨，美元兑人民币上涨。

黄金 -120
原因：积存金价格小幅回调。

英伟达 +290
原因：美股AI芯片小仓上涨。
```

## AI 建议优先级

AI建议不再全部同等权重，改成行动队列，并下沉到持仓页。

优先级：

```text
★★★★★ 必须处理
★★★★ 建议关注
★★ 提醒
```

示例：

```text
★★★★★ 必须处理
黄金仓位偏高，建议暂停新增。

★★★★ 建议关注
半导体基金近6个月弱于同板块平均，加入观察名单。

★★ 提醒
QDII今日收益为估算值，不要按单日估算频繁操作。
```

## 同板块基金对比表达方式

为了避免系统过度引导频繁换基，表达方式从“建议同板块替换”弱化为：

- 加入观察名单
- 存在更优候选基金
- 建议进一步比较
- 点击查看同板块对比

只有同时满足多个条件时，才提示：

```text
建议评估换基
```

触发条件包括：

- 长期收益明显落后
- 费率偏高
- 跟踪误差明显更差
- 基金规模过小

## QDII专区

QDII专区明确区分：

- 估算收益
- 确认收益
- T+2净值延迟
- 汇率影响
- 最终以基金公司公布净值为准

计算逻辑：

```text
估算收益 = 持有份额 × 估算净值 - 持仓成本
确认收益 = 持有份额 × 最新确认净值 - 持仓成本
估算净值 = 最新确认净值 × (1 + 对应指数涨跌) × (1 + 汇率变化修正)
```
